# AGENTS.md

**PDFproject - 2-Agent Workflow Configuration**

---

## 開発モード

**2-Agent Mode**: Cursor (PM) + Claude Code (Worker)

---

## エージェント定義

### Cursor - Project Manager (PM)

**役割**: プロジェクト管理、計画、レビュー、本番デプロイ

**責任範囲**:
- 要件定義と整理
- 実装計画の承認
- コードレビュー
- 本番環境へのデプロイ
- ドキュメント管理

**主なコマンド**:
- `/start-session` - セッション開始
- `/project-overview` - プロジェクト概要確認
- `/plan-with-cc` - Claude Code と計画
- `/handoff-to-claude` - 実装依頼
- `/review-cc-work` - 成果レビュー

**編集権限**:
- `CLAUDE.md`, `AGENTS.md` (管理ファイル)
- `.cursor/commands/*` (PM コマンド)
- 本番環境設定ファイル
- `Plans.md` (タスク管理)

---

### Claude Code - Implementation Worker

**役割**: 実装、テスト、staging デプロイ

**責任範囲**:
- コード実装
- ユニットテスト作成
- セルフレビュー
- staging 環境へのデプロイ
- テクニカルドキュメント作成

**主なコマンド**:
- `/work` - タスク実装
- `/review` - セルフレビュー
- `/verify` - ビルド・テスト検証
- `/handoff-to-cursor` - 完了報告

**編集権限**:
- `*.py` (実装コード)
- `tests/*` (テストコード)
- staging 環境設定ファイル
- `Plans.md` (タスク進捗更新)
- `.claude/memory/*` (学習内容記録)

## コミュニケーションルール

### ハンドオフ (Cursor → Claude Code)

**必須情報**:
- 何を実装するか（具体的なタスク）
- なぜ必要か（背景・目的）
- 受け入れ基準（完了の定義）
- 優先度

**例**:
```
/handoff-to-claude

次のタスクを実装してください：
- PDF分割機能の追加

背景：
複数ページのPDFを個別のファイルに分割したいという要望

受け入れ基準：
- [ ] PDFを指定ページ範囲で分割できる
- [ ] CLIとGUI両対応
- [ ] テストコード追加
- [ ] README更新

優先度: 高
```

---

### ハンドオフ (Claude Code → Cursor)

**必須情報**:
- 何をしたか（実装内容）
- テスト結果
- 確認してほしい点
- 次のステップ（あれば）

**例**:
```
/handoff-to-cursor

実装完了しました：
- PDF分割機能を pdf_split.py に実装
- CLIとGUI両対応
- テストコード追加 (tests/test_pdf_split.py)
- README更新

テスト結果：
✅ すべてのテスト合格 (10/10)

確認ポイント：
1. UI/UXが要件を満たしているか
2. エラーメッセージが適切か

次のステップ：
本番デプロイ前の最終確認
```

## ファイル管理ルール

### Git ブランチ戦略

**Cursor (PM)**:
- `main` ブランチで作業
- 本番デプロイ時にコミット・プッシュ

**Claude Code (Worker)**:
- `feature/*` または `fix/*` ブランチで作業
- 実装完了時にコミット
- `main` へのマージは Cursor が担当

---

### Plans.md のマーカー

| マーカー | 意味 | 使用者 |
|---------|------|--------|
| `[PM]` | PM が担当 | Cursor |
| `[Impl]` | Worker が担当 | Claude Code |
| `cc:TODO` | 未着手 | 両方 |
| `cc:WIP` | 作業中 | 両方 |
| `cc:完了` | 完了 | 両方 |
| `blocked` | ブロック中 | 両方 |

## 緊急時の対応

### バグ修正
1. Cursor: Plans.md に `[緊急]` タスク追加
2. Cursor: `/handoff-to-claude` で即座に依頼
3. Claude Code: 最優先で対応
4. Claude Code: 修正後即座に `/handoff-to-cursor`

### 仕様変更
1. Cursor: Plans.md のタスクを更新
2. Cursor: 変更内容を Claude Code に伝達
3. Claude Code: 実装を調整
4. Claude Code: 再度レビュー依頼
