プロジェクト概要

PDFproject は、画像のPDF化からPDF分割・結合・編集・ページ番号付与・バッチ処理までを一つに揃えた CLI/GUI ツール集です。すべて Python で実装されており、各機能は独立した1ファイルで構成されています。

項目内容
言語Python 3.13+
主要ライブラリPillow / pypdf / reportlab / tkinter
テストpytest（全112テスト成功）
動作環境macOS（GUI は tkinter 標準）
ライセンスMIT

インストール

# 依存ライブラリをまとめてインストール
pip install -r requirements.txt

# 個別にインストールする場合
pip install pypdf reportlab Pillow pytest
💡 macOSでは tkinter は標準で含まれます。表示されない場合は brew reinstall python@3.13 で再インストール。

6つのツール

1. img2pdf.py — 画像→PDF変換

PNG/JPEGの画像群を1つのPDFにまとめます。透過PNGは白背景に合成。

python3 img2pdf.py --select-files                # GUI
python3 img2pdf.py page1.png page2.jpg -o out.pdf
python3 img2pdf.py ./screenshots -o report.pdf
python3 img2pdf.py ./pages --recursive -o book.pdf

主なオプション: -o 出力先 / --recursive サブフォルダ含む / -q 画質(1-100, 既定95) / --verbose

2. pdf_splitter.py — PDF分割

ページ範囲指定で複数ファイルに分割します。

python3 pdf_splitter.py --select-files
python3 pdf_splitter.py input.pdf --ranges "1-3,4-7,8-10" -o ./split
python3 pdf_splitter.py document.pdf --ranges "5" -o ./pages

範囲指定: 1-3 / 1:3 / 5 / 1-3,5,7-9

3. pdf_merger.py — PDF結合

複数のPDFを1つに結合（最低2ファイル必須）。入力順序がそのまま結合順序になります。

python3 pdf_merger.py --select-files
python3 pdf_merger.py file1.pdf file2.pdf -o merged.pdf
python3 pdf_merger.py chapter*.pdf -o book.pdf

4. pdf_edit.py — ページ削除・並べ替え・挿入

1つのコマンドで複数操作を組み合わせ可能です。

# 削除
python3 pdf_edit.py input.pdf --delete "1,3,5-7" -o out.pdf

# 並べ替え（全ページ指定が必要）
python3 pdf_edit.py input.pdf --reorder "3,1,2,4-10" -o out.pdf

# 別PDFから挿入: source.pdf:pages@position
python3 pdf_edit.py input.pdf --insert "cover.pdf:1@1" -o out.pdf

# 複数操作の組み合わせ
python3 pdf_edit.py input.pdf --delete "1,5" --reorder "2,1,3-10" -o out.pdf

5. pdf_page_number.py — ページ番号挿入

6つの位置指定とカスタムフォーマットに対応。

python3 pdf_page_number.py input.pdf -o out.pdf \
  --position bottom-right \
  --format "{page}/{total}"

# 表紙をスキップ
python3 pdf_page_number.py book.pdf -o numbered.pdf --skip-pages "1,2"

# 日本語フォーマット
python3 pdf_page_number.py input.pdf -o out.pdf --format "第{page}ページ"
位置値
上部 左 / 中央 / 右top-left / top-center / top-right
下部 左 / 中央 / 右bottom-left / bottom-center / bottom-right

6. pdf_batch.py — バッチ処理 NEW

ディレクトリ内の全PDFに対し同一操作を一括実行。最後に成功/失敗のサマリーを表示します。

# 一括ページ番号追加
python3 pdf_batch.py add-page-numbers \
  --input-dir ./pdfs --output-dir ./numbered \
  --position bottom-right --format "{page}/{total}"

# 一括分割
python3 pdf_batch.py split \
  --input-dir ./pdfs --output-dir ./split --ranges "1-5,6-10"

# 一括ページ削除
python3 pdf_batch.py delete \
  --input-dir ./pdfs --output-dir ./edited --delete-pages "1"

よくある使い方パターン

パターン1: スキャン画像をPDFにまとめる

python3 img2pdf.py ./scanned_images -o document.pdf

パターン2: 大きなPDFを章ごとに分割

python3 pdf_splitter.py book.pdf --ranges "1-20,21-50,51-100" -o ./chapters

パターン3: 複数PDFを1冊の本に結合

python3 pdf_merger.py cover.pdf intro.pdf chapter1.pdf chapter2.pdf -o book.pdf

パターン4: 不要ページの削除と並べ替え

python3 pdf_edit.py presentation.pdf --delete "2,5,8-10" -o cleaned.pdf

パターン5: ページ番号を追加（表紙はスキップ）

python3 pdf_page_number.py document.pdf -o numbered.pdf \
  --skip-pages "1" --position bottom-right --format "{page}/{total}"

実践例: 論文PDFのフル組み立て

# 1. フィギュア画像をPDFに
python3 img2pdf.py ./figures -o figures.pdf

# 2. 章PDFと結合
python3 pdf_merger.py cover.pdf abstract.pdf intro.pdf methods.pdf figures.pdf discussion.pdf -o draft.pdf

# 3. 不要ページ削除
python3 pdf_edit.py draft.pdf --delete "15,20-22" -o cleaned.pdf

# 4. ページ番号付与（表紙・目次スキップ）
python3 pdf_page_number.py cleaned.pdf -o final.pdf \
  --skip-pages "1,2" --position bottom-center --format "- {page} -"

Web版エディタ (pdf_editor_app)

ブラウザで動作するPDFエディタ。StartEditor.command をダブルクリックするか、以下のコマンドで起動できます。

python3 run_editor.py
# → ブラウザで http://localhost:5000 などにアクセス

構成: Flask ベースのアプリで、api.py（REST API）、annotation_engine.py（注釈処理）、pdf_operations.py（PDF操作）、static/（フロントエンド）から成ります。

トラブルシューティング

ModuleNotFoundError: No module named 'pypdf'

pip install -r requirements.txt

tkinter not available for GUI mode

brew reinstall python@3.13

File is not a valid PDF

元のPDFが破損している可能性があります。Adobe Acrobat Reader等で開けるか確認してください。

出力ファイルが作られない

  1. 出力ディレクトリが存在するか: mkdir -p ./output
  2. 書き込み権限があるか: ls -la ./output
  3. ディスク容量: df -h
  4. --verbose で詳細ログを確認
